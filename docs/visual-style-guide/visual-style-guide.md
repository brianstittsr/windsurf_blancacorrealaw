# Visual Style Guide
# Blanca Correa Law Immigration Website

## 1. Introduction

### 1.1 Purpose

This visual style guide defines the design standards for the Blanca Correa Law Immigration Website. It provides comprehensive guidelines for implementing the "Pathways to Belonging" visual theme across all aspects of the website. This document should be used by designers, developers, and content creators to ensure a consistent, accessible, and engaging user experience that reflects the firm's brand values and meets the needs of its target audience.

### 1.2 Brand Essence

**Brand Promise:** "Your Bridge to Immigration Success in North Carolina"

**Brand Values:**
- Expertise with empathy
- Cultural authenticity
- Clarity and transparency
- Supportive guidance
- Local knowledge

### 1.3 Visual Theme: "Pathways to Belonging"

The "Pathways to Belonging" visual theme is built around the metaphor of journeys, pathways, and bridges - representing the immigration journey while avoiding clichéd imagery. This theme visually communicates:

- The journey from uncertainty to belonging
- Clear pathways through complex processes
- Bridges connecting cultures and experiences
- Warmth and hope throughout the immigration journey
- Professional guidance with human connection

### 2.4 Color Accessibility

All color combinations must meet WCAG 2.1 AA standards for contrast:
- Text and background combinations must have a minimum contrast ratio of 4.5:1
- Large text (18pt+) must have a minimum contrast ratio of 3:1
- UI components and graphical objects must have a minimum contrast ratio of 3:1

### 2.5 Color Application Guidelines

- Use the primary Terracotta color sparingly for maximum impact
- Deep Teal provides a professional foundation for the design
- Sunrise Yellow should be used for highlights and to draw attention
- Maintain consistent color usage across both English and Spanish versions
- Ensure color is never the sole means of conveying information (for accessibility)

### 3.3 Multilingual Typography Considerations

- Both typefaces have excellent multilingual character support
- Maintain consistent visual hierarchy across both languages
- Allow for text expansion in Spanish (typically 15-25% longer than English)
- Ensure line heights accommodate accented characters
- Test readability in both languages at all screen sizes

### 3.4 Typography Guidelines

- Maintain consistent spacing between headings and body text
- Use sentence case for headings (capitalize first word only)
- Limit use of all caps to small UI elements or labels
- Ensure adequate contrast between text and background
- Use bold sparingly for emphasis within body text

## 4. Imagery

### 4.1 Photography Style

The website will use authentic photography featuring real people and local landmarks with these characteristics:

- **Authenticity:** Real people in natural settings, not posed stock photography
- **Diversity:** Representation of various cultures, ages, and backgrounds
- **Warmth:** Warm lighting that creates a welcoming, hopeful atmosphere
- **Local Context:** Incorporation of North Carolina settings and landmarks
- **Journey Elements:** Visual elements that suggest pathways, progress, and connection

### 4.2 Photography Categories

| Category | Purpose | Examples |
|----------|---------|----------|
| Hero Images | Create emotional connection | Families reunited, successful clients, welcoming scenes |
| Journey Imagery | Illustrate progress | Paths, bridges, doorways, symbolic journey elements |
| Portrait Photography | Build trust | Attorney portraits, client testimonial photos |
| Documentary Style | Show authenticity | Real client interactions, office environment, community events |
| Local Landmarks | Establish local expertise | NC-specific locations, USCIS offices, courthouses |

### 4.3 Image Technical Requirements

- High-resolution (minimum 72dpi for web)
- Responsive image implementation with multiple sizes
- Optimized file sizes for fast loading (particularly on mobile)
- Alt text for all images
- Proper cropping for different viewport sizes

### 4.4 Image Usage Guidelines

- Avoid clichéd immigration imagery (gavels, flags, handshakes)
- Ensure diverse representation across the website
- Use consistent editing style for color temperature and tone
- Pair images with relevant, meaningful content
- Respect privacy and obtain proper permissions for client photos

## 5. Visual Elements

### 5.1 Path/Journey Motif

The path/journey motif is a core visual element that should be integrated throughout the design:

- **Path Elements:** Curved or straight lines suggesting direction and movement
- **Connection Points:** Visual elements showing progress along a journey
- **Bridges:** Symbolic elements connecting different points or concepts
- **Gateways:** Visual frames suggesting transitions or new opportunities
- **Milestones:** Visual markers indicating progress or achievement

### 5.2 Icons

Icons should follow these guidelines:

- **Style:** Simple, clear line icons with consistent stroke weight
- **Size:** 24px standard size (scale appropriately for different contexts)
- **Color:** Primary or neutral colors from the defined palette
- **Consistency:** Use from a single icon family for visual coherence
- **Meaning:** Icons should have clear meaning and be accompanied by text labels

### 5.3 Icon Categories

| Category | Purpose | Examples |
|----------|---------|----------|
| Navigation Icons | Guide user through site | Home, search, menu, language toggle |
| Pathway Icons | Represent immigration types | Family, employment, citizenship, asylum |
| Process Icons | Show steps or actions | Document, calendar, checklist, upload |
| Status Icons | Indicate state or progress | Complete, in progress, waiting, alert |
| Service Icons | Represent service offerings | Consultation, document review, representation |

### 5.4 Visual Process Maps

Process maps are a key visual element for explaining immigration pathways:

- **Style:** Clean, linear progression with clear connection points
- **Components:** Steps, decision points, timelines, and outcomes
- **Interaction:** Interactive elements that reveal additional information
- **Responsiveness:** Adapt from horizontal flow (desktop) to vertical flow (mobile)
- **Color Coding:** Consistent use of color to indicate different phases or requirements

## 6. Layout & Grid System

### 6.1 Grid Structure

The website will use a responsive 12-column grid system:

- **Desktop:** 1200px max-width container with 30px gutters
- **Tablet:** 768px-1199px with 20px gutters
- **Mobile:** 320px-767px with 16px gutters
- **Margins:** Minimum 16px on mobile, scaling up to 60px on desktop

### 6.2 Spacing System

A consistent spacing system based on 8px increments:

| Token | Value | Usage |
|-------|-------|-------|
| space-xs | 4px | Minimal spacing, icon padding |
| space-sm | 8px | Tight spacing, between related elements |
| space-md | 16px | Standard spacing, margins and padding |
| space-lg | 24px | Generous spacing, section separators |
| space-xl | 32px | Large spacing, major section breaks |
| space-xxl | 48px | Extra large spacing, page sections |
| space-xxxl | 64px | Maximum spacing, page breaks |

### 6.3 Layout Patterns

| Pattern | Purpose | Usage |
|---------|---------|-------|
| Single Column | Focus on content | Mobile views, focused content pages |
| Two Column | Content with sidebar | Resource pages, blog posts |
| Three Column | Multiple options | Service comparisons, resource listings |
| Card Grid | Visual browsing | Success stories, resource directory |
| Z-Pattern | Narrative flow | Homepage, landing pages |
| F-Pattern | Information scanning | Text-heavy pages, FAQs |

### 6.4 Responsive Behavior

- Mobile-first approach to design and development
- Consistent component behavior across breakpoints
- Strategic content prioritization on smaller screens
- Touch-friendly tap targets (minimum 44x44px) on mobile
- Consideration for both portrait and landscape orientations

## 7. Components

### 7.1 Navigation Components

#### 7.1.1 Primary Navigation

- **Desktop:** Horizontal navigation bar with dropdown menus
- **Mobile:** Hamburger menu with expandable sections
- **States:** Normal, hover, active, current
- **Features:** Language toggle, search, urgent help button

#### 7.1.2 Language Toggle

- **Style:** Text labels ("EN"/"ES") with subtle visual indicator
- **States:** Current language highlighted
- **Position:** Consistent placement in top navigation
- **Behavior:** Maintains page context when switching languages

#### 7.1.3 Breadcrumbs

- **Style:** Text links separated by subtle dividers
- **Behavior:** Show path from home to current page
- **Responsiveness:** May be hidden on mobile views
- **Language:** Updates based on selected language

### 7.2 Button Components

#### 7.2.1 Primary Buttons

- **Style:** Solid Terracotta background with white text
- **States:** Normal, hover, active, disabled
- **Sizes:** Small, medium (default), large
- **Shape:** Slightly rounded corners (4px radius)
- **Icons:** Optional left or right aligned icon

#### 7.2.2 Secondary Buttons

- **Style:** Deep Teal outline with Deep Teal text
- **States:** Normal, hover, active, disabled
- **Sizes:** Small, medium (default), large
- **Shape:** Matches primary button radius
- **Icons:** Optional left or right aligned icon

#### 7.2.3 Tertiary Buttons

- **Style:** No border, Deep Teal text with subtle underline
- **States:** Normal, hover, active, disabled
- **Sizes:** Small, medium (default), large
- **Icons:** Optional left or right aligned icon

### 7.3 Card Components

#### 7.3.1 Pathway Cards

- **Purpose:** Present immigration pathway options
- **Content:** Icon, title, brief description, action button
- **Style:** White background, subtle shadow, accent color top border
- **Interaction:** Entire card clickable with appropriate hover state

#### 7.3.2 Resource Cards

- **Purpose:** Display resources in directory
- **Content:** Title, description, metadata (location, languages, services)
- **Style:** White background, minimal border, category indicator
- **Interaction:** Expandable for additional details

#### 7.3.3 Testimonial Cards

- **Purpose:** Showcase client success stories
- **Content:** Quote, client photo, name, immigration type
- **Style:** Subtle background, prominent quote styling
- **Variation:** With/without photo options

### 7.4 Form Components

#### 7.4.1 Text Inputs

- **Style:** Clear borders, adequate padding, visible focus state
- **States:** Default, focus, filled, error, disabled
- **Variations:** Single line, multi-line, with icon, with clear button
- **Labels:** Always visible, clear positioning

#### 7.4.2 Selection Controls

- **Radio Buttons:** For single selection from limited options
- **Checkboxes:** For multiple selections or single toggles
- **Dropdowns:** For single selection from many options
- **States:** Default, selected, hover, focus, disabled

#### 7.4.3 Form Layout

- **Grouping:** Logical grouping of related fields
- **Progression:** Clear visual hierarchy and flow
- **Validation:** Inline validation with helpful error messages
- **Responsive:** Stacks appropriately on smaller screens

### 7.5 Interactive Tool Components

#### 7.5.1 Progress Indicators

- **Style:** Clear visual representation of progress through multi-step process
- **States:** Complete, current, upcoming
- **Responsiveness:** Simplified on mobile but maintains context
- **Animation:** Subtle transitions between states

#### 7.5.2 Assessment Questions

- **Layout:** Clear question with response options below
- **Interaction:** Immediate visual feedback on selection
- **Navigation:** Clear next/previous controls
- **Help:** Contextual help available for complex questions

#### 7.5.3 Results Displays

- **Style:** Visual confidence indicators (progress bars, percentages)
- **Organization:** Hierarchical presentation of results
- **Actions:** Clear next steps and related resources
- **Saving:** Options to save, print, or share results

## 8. Animation & Interaction

### 8.1 Micro-interactions

- **Purpose:** Provide feedback, guide attention, enhance engagement
- **Duration:** Quick (200-300ms) for interface feedback
- **Timing:** Ease-in-out for natural feeling
- **Usage:** Button states, form interactions, navigation changes

### 8.2 Page Transitions

- **Style:** Subtle fade or slide transitions between pages
- **Duration:** Medium (300-500ms) for page transitions
- **Purpose:** Maintain context and reduce cognitive load
- **Performance:** Optimized for performance, especially on mobile

### 8.3 Content Animations

- **Purpose:** Reveal information progressively, highlight key points
- **Style:** Subtle entrance animations for content sections
- **Trigger:** Typically on scroll or view
- **Accessibility:** Respects reduced motion preferences

### 8.4 Interactive Elements

- **Hover States:** Clear visual feedback on interactive elements
- **Focus States:** Distinct, visible focus indicators for keyboard navigation
- **Active States:** Immediate feedback when elements are activated
- **Loading States:** Clear indication when content or actions are loading

## 9. Accessibility Standards

### 9.1 Color & Contrast

- All text meets WCAG 2.1 AA contrast requirements (4.5:1 for normal text, 3:1 for large text)
- Interactive elements have sufficient contrast against their backgrounds (minimum 3:1)
- Color is never the sole means of conveying information
- Text over images has sufficient contrast through proper image selection or overlays

### 9.2 Typography & Readability

- Text can be resized up to 200% without loss of content or functionality
- Line heights are adequate for readability (minimum 1.5 for body text)
- Letter spacing is optimized for readability
- Paragraphs have adequate spacing
- Maximum line length is controlled for readability (66 characters recommended)

### 9.3 Navigation & Interaction

- All interactive elements are keyboard accessible
- Focus order is logical and follows visual layout
- Focus indicators are clearly visible
- Touch targets are adequately sized (minimum 44x44px)
- Critical functions have multiple interaction methods

### 9.4 Content & Structure

- Proper heading hierarchy is maintained
- Landmark regions are properly defined
- Images have appropriate alternative text
- Form elements have associated labels
- Tables have proper headers and structure

```css
:root {
  /* Colors: Primary */
  --color-terracotta: #E07A5F;
  --color-deep-teal: #3D5A80;
  --color-sunrise-yellow: #F9C74F;
  
  /* Colors: Neutral */
  --color-warm-white: #F8F5F2;
  --color-light-gray: #E5E5E5;
  --color-medium-gray: #9D9D9D;
  --color-dark-gray: #4A4A4A;
  --color-charcoal: #2D2D2D;
  
  /* Colors: Functional */
  --color-success: #81B29A;
  --color-alert: #E76F51;
  --color-info: #457B9D;
  
  /* Typography */
  --font-heading: 'Montserrat', 'Nunito', Arial, sans-serif;
  --font-body: 'Source Serif Pro', Georgia, 'Times New Roman', serif;
  
  /* Spacing */
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --space-xxl: 48px;
  --space-xxxl: 64px;
  
  /* Border Radius */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  
  /* Shadows */
  --shadow-sm: 0 1px 3px rgba(0,0,0,0.12);
  --shadow-md: 0 4px 6px rgba(0,0,0,0.1);
  --shadow-lg: 0 10px 15px rgba(0,0,0,0.1);
  
  /* Transitions */
  --transition-fast: 200ms ease-in-out;
  --transition-medium: 300ms ease-in-out;
  --transition-slow: 500ms ease-in-out;
}
```

### 10.2 Component Library Implementation

- Implement components as reusable React components
- Document component props and variants
- Include accessibility requirements in component specifications
- Ensure responsive behavior is built into each component
- Test components in both languages

### 10.3 Asset Management

- Optimize images for web delivery
- Implement responsive images with appropriate srcset attributes
- Use SVG for icons and simple illustrations
- Establish naming conventions for all assets
- Create both English and Spanish versions of text-containing graphics

### 10.4 Documentation

- Maintain living documentation of the design system
- Include code examples with each component
- Document accessibility requirements and implementation
- Provide guidance for content creators on visual elements
- Update documentation as the design system evolves
